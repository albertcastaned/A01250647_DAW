Definiciones:

  • URL:

    Un URL o "Uniform Resource Location" describe la dirección en la que esta un recurso en internet, ademas de el tipo de recurso que es.

  • MIME:

    El protocolo MIME o "Multi-purpose Internet Mail Extensions" es una extension del protocolo de email regular requerido si el texto a enviar no esta en ASCII. [1]

  • HTTP:

    El protocolo HTTP consiste en diferentes métodos de peticiones para obtener y enviar información web. Algunos de ellos son POST para enviar datos en forma de formulario, GET para leer una representación de un recurso, PUT para actualizar o crear contenidos similar a POST con la diferencia que se ignoraran peticiones repetidas, DELETE para eliminar un recurso, y el HEAD que pide el código de respuesta. [2]

    Algunos códigos de error comunes son el 404 que es cuando una petición GET obtiene una respuesta no encontrada o inaccesible, 401 y 403 cuando no se tiene permiso para la petición, 500 para un error interno en el servidor, y 504 cuando se tiene una conexión lenta. [3]

  • XML:

    XML es un "markup language", un tipo de lenguaje que describe un documento o una estructura de datos junto a su organización. Utiliza etiquetas para clasificar, categorizar, y organizar información. Es util para intercambiar información entre diferentes dispositivos. Los dos tipos de métodos mas populares para XML son Document Type Definitions (DTD) el cual es util para definir la estructura del XML con mucho texto, y XML Schemas para definir los bloques de elementos que tendrá el XML. [4]

  • XHTML:

    XHTML es una reconstrucción de HTML basado en las reglas y sintaxis de XML, algunos lo consideran el futuro del desarrollo web. [5]

  • HTML5:

    HTML5 es la version mas moderna de HTML el cual es un lenguaje que configura como se desplegá la información en un navegador web.

  • JavaScript:

    JavaScript es un lenguaje de programación de alto nivel para la navegación web, comúnmente utilizado para agregar interacción con el usuario en una pagina web.

  • PHP:

    Acronimo de PHP: Hypertext Preprocessor. Es un lenguaje de programacion multi-uso especialmente utilizado para el desarollo web ya que puede ser insertado en HTML. [6]

  • JavaServer Pages:

    JavaServer Pages es una tecnología hecha en Java para crear aplicaciones web dinámicas. [7]

  • AJAX:

    AJAX (Asynchronous JavaScript and XML) es una técnica que permite crear aplicaciones web mas interactivas usando HTML, XML, CSS, y JavaScript. [8]


Cuestionario:

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    El internet se refiere a la inmensa interconexión de computadoras alrededor del mundo mientras la World Wide Web (WWW) es la colección de paginas web que tiene la red de computadoras.

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • GET: Recuperar recursos de un sitio web.
    • HEAD: Recupera el código de respuesta.
    • POST: Envia una solicitud con datos.
    • PUT: Similar al POST excepto que varias peticiones repetidas seran ignoradas.
    • PATCH: Aplica modificaciones ligeras a recursos existentes.
    • DELETE: Borra uno de los recursos existentes.

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    El metodo POST, ya que esta se encarga de enviar datos del cliente hacia el servidor para despues esperar una respuesta.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    Se requiere el método GET para obtener la información HTML del servidor para poder asi desplegar su contenido en pantalla.

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    El código de respuesta indica que no hubo ningún error y fue una petición exitosa.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    Depende de la dirección que el usuario intento ingresar. Si el usuario intento entrar a una pagina web que debería ser accesible para el entonces los desarrolladores deberán verificar porque no se pudo acceder. De lo contrario, significa querido el usuario se equivoco al ingresar la dirección y por lo tanto intento entrar a una pagina no existente.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Si, ya que este indica un problema interno en el servidor que debe ser arreglado por el desarrollador.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    Significa que el atributo fue descontinuado, o actualizado con otro nombre. Algunos ejemplos de elementos despreciados en HTML5 son: [9]
    • Los atributos de <font>:Atributos como color, face, y size son remplazados con el uso de CSS.
    • Los atributos de <table>: Atributos como border, cellpadding, y cellspacing, remplazados con el uso de CSS.
    • Los atributos de <th> y <td>: Atributos como align y valign remplazados con el uso de CSS.

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    • Nuevos elementos semánticos como section, article, aside, header, footer, nav, etc...
    • Nuevos tipos de input como datetime, date, month, week, number sin el uso de JS.
    • Posibilidad de utilizar XML para expresar laa sintaxis (XHTML).

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    [10]

    • td: Define una celda de la tabla.

    • th: Define el encabezado de la tabla.

  • ¿Cuál es el ciclo de vida de los sistemas de información?

    [11]
    1. Investigación preliminar: Identificar problemas a resolver
    2. Análisis detallado de la situación: Comprensión total de la información que necesita el cliente y como es el proceso de su trabajo.
    3. Diseño de nuevo sistema: Enlistar los elementos fundamentales para resolver los problemas del cliente.
    4. Desarrollo y documentación del sistema: Elaborar software y documentar el uso de el.
    5. Mantenimiento del sistema y conocimientos: Adaptar el sistema a los cambios del entorno y los requeridos para seguir con vida.

  • ¿Cuál es el ciclo de desarrollo de sistemas de información?

    [12]
    1. Planning: Investigar el problema y determinar soluciones.
    2. Análisis de Sistemas y Requisitos: Se determinan los requisitos funcionales.
    3. Diseño de Sistema: Detallar las especificaciones, funciones, y operaciones necesarias para satisfacer los requisitos.
    4. Desarollo: Inicia proceso de producción de codificación, y documentación del software.
    5. Integración y Testing: Integrar los módulos del software y hacer pruebas de casos de uso para determinar si el diseño satisface las necesidades.
    6. Implementación: El proceso de codificación del software.
    7. Operaciones y Mantenimiento: Últimos ajustes y agregación de funciones requeridas por el cliente.
