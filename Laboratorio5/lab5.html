Cuestionario

  • ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    Es importante hacer una validacion propio para verificar que el usuario no ingrese datos inadecuados por error, ademas de transformar los datos a otro formato si es necesario

  • ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    Eliminando la funcion JS que valida los datos de donde se llama, como el boton. Esto se puede lograr desde la consola.

  • ¿Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    Porque aunque es necesario validar desde el lado del servidor, las validaciones desde el lado del cliente nos sigue facilitando evitar errores y darle retroalimentacion inmediata al usuario.
