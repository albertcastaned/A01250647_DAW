Cuestionario

¿Por qué es importante hacer un session_unset() y luego un session_destroy()?

Para borrar las variables de la session, y los datos de la sesion respectivamente. Session_unset solo borra las variables, mientras session_destroy borra los datos de la sesion pero no las las variables.

¿Cuál es la diferencia entre una variable de sesión y una cookie?

Los datos de la sesion se almacenan en el servidor. En cambio, los datos de una cookie se almacena en el navegador del cliente o usuario. Las sesiones utilizan cookies, pero estas se almacenan en el registro del usuario en el servidor.

¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?

Existen varias tecnicas para evitar que un archivo subido tenga el mismo nombre que otro. La principal utilizada es Hashing, el cual utiliza un algoritmo predefinido que encripta el nombre del archivo para garantizar unicidad. Tambien, se puede utilizar timestamps que asigna el tiempo de subida al archivo. Se puede realizar una combinacion de las 2 tecnicas anteriores.

¿Qué es CSRF y cómo puede prevenirse?

Un tipo de ataque que hace que usuarios autenticados logren acceder a funciones o acciones que no deberian poder realizarse con su rol. Una forma de prevenir este ataque es asignar un Cookie con un valor generado aleatoriamente cuando un usuario inicia sesion en una pagina. Al enviar peticiones con formas, el servidor compara si el Cookie que es enviado junto a la forma corresponde al del usuario.
