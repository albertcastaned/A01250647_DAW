Definiciones:

  • URL:

    Un URL o "Uniform Resource Location" describe la dirección en la que esta un recurso en internet, ademas de el tipo de recurso que es.

  • MIME:

    El protocolo MIME o "Multi-purpose Internet Mail Extensions" es una extension del protocolo de email regular requerido si el texto a enviar no esta en ASCII. [1]

  • HTTP:

    El protocolo HTTP consiste en diferentes métodos de peticiones para obtener y enviar información web. Algunos de ellos son POST para enviar datos en forma de formulario, GET para leer una representación de un recurso, PUT para actualizar o crear contenidos similar a POST con la diferencia que se ignoraran peticiones repetidas, DELETE para eliminar un recurso, y el HEAD que pide el código de respuesta. [2]

    Algunos códigos de error comunes son el 404 que es cuando una petición GET obtiene una respuesta no encontrada o inaccesible, 401 y 403 cuando no se tiene permiso para la petición, 500 para un error interno en el servidor, y 504 cuando se tiene una conexión lenta. [3]

  • XML:

    XML es un "markup language", un tipo de lenguaje que describe un documento o una estructura de datos junto a su organización. Utiliza etiquetas para clasificar, categorizar, y organizar información. Es util para intercambiar información entre diferentes dispositivos. Los dos tipos de métodos mas populares para XML son Document Type Definitions (DTD) el cual es util para definir la estructura del XML con mucho texto, y XML Schemas para definir los bloques de elementos que tendrá el XML. [4]

  • XHTML:

    XHTML es una reconstrucción de HTML basado en las reglas y sintaxis de XML, algunos lo consideran el futuro del desarrollo web. [5]

  • HTML5:

    HTML5 es la version mas moderna de HTML el cual es un lenguaje que configura como se desplegá la información en un navegador web.

  • JavaScript:

    JavaScript es un lenguaje de programación de alto nivel para la navegación web, comúnmente utilizado para agregar interacción con el usuario en una pagina web.

  • PHP:

    Acronimo de PHP: Hypertext Preprocessor. Es un lenguaje de programacion multi-uso especialmente utilizado para el desarollo web ya que puede ser insertado en HTML. [6]

  • JavaServer Pages:

    JavaServer Pages es una tecnología hecha en Java para crear aplicaciones web dinámicas. [7]

  • AJAX:

    AJAX (Asynchronous JavaScript and XML) es una técnica que permite crear aplicaciones web mas interactivas usando HTML, XML, CSS, y JavaScript. [8]

Cuestionario:

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    Se debe tener cuidado al usarlo, ya que anularía atributos en otras secciones de la misma clase debido a que !important tiene preferencia.

  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    Se debe tener cuidado ya que puede resultar pixeleado si no se encoge a una resolución adecuada.

  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    Dependiendo del contexto. % es util para cuando necesitamos que un elemento este en un tamaño en relacion con otro, mientras px es usualmente la unidad estandar en web. Pt es utilizado para impresiones.

  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    Es mas rápido ya que se ignora el espacio en blanco de un CSS normal.
